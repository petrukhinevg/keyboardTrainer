# Frontend Roadmap

## Цель

Собрать frontend для SQL-тренажёра с входом, картой уровней, экраном задания, feedback и экраном прогресса.

## Правило использования roadmap

- Пункты ниже предназначены для создания GitHub Issues.
- Одна задача должна давать завершённый и проверяемый frontend-результат.
- Если frontend-пункт становится слишком крупным, его нужно оформлять как крупную `parent issue` и делить на `sub-issues`.

## Базовые UI-ограничения

- Приложение должно быть одностраничным (`SPA`).
- Основной рабочий экран строится в три колонки.
- Слева находится вертикальная панель выбора уровней и задач.
- В центре находится область с описанием урока и условиями задачи.
- Справа находится рабочая область решения:
  - сверху окно ввода SQL-запроса;
  - снизу зона результатов выполнения;
  - нижняя зона делится пополам между блоками результата.

## MVP для frontend

MVP frontend считается готовым, когда пользователь может:

1. войти в приложение;
2. увидеть доступные уровни;
3. открыть задание;
4. отправить SQL-запрос;
5. получить feedback;
6. увидеть обновлённый прогресс.
