# Backend Roadmap

## Цель

Собрать backend для SQL-тренажёра с авторизацией, выдачей заданий, проверкой SQL и сохранением прогресса.

## Правило использования roadmap

- Пункты ниже предназначены для создания GitHub Issues.
- Одна задача должна давать завершённый и проверяемый backend-результат.
- Если backend-пункт становится слишком крупным, его нужно оформлять как крупную `parent issue` и делить на `sub-issues`.

## Примечание по seed data

- `Seed data` в этом roadmap означает стартовый, заранее подготовленный набор уровней и задач.
- Это не генерация уровней и не процедурное создание контента.
- Такие данные нужны, чтобы MVP можно было запускать, тестировать и демонстрировать до появления отдельного редактора контента или генератора.

## MVP для backend

MVP backend считается готовым, когда API умеет:

1. идентифицировать пользователя;
2. отдавать список уровней и задач;
3. принимать SQL-ответ;
4. проверять результат выполнения;
5. сохранять прогресс и попытки;
6. открывать следующий контент.
