# Roadmap

## Цель проекта

Создать приложение для тренировки SQL-навыков через последовательные уровни и задания.

## Правило использования roadmap

- Roadmap описывает крупные `parent issues` и компактные `sub-issues` для GitHub.
- Одна задача должна быть логически завершённой и проверяемой.
- Если пункт получается слишком большим для одной задачи, его нужно заводить как крупную `parent issue` и дробить на `sub-issues`.
- Формулировки ниже даны так, чтобы по ним можно было сразу создавать GitHub Issues.
- Необходимо выполнять вначале таски бэка, а затем соответствующие таски фронта. не переходить на следующую таску бэка, если на фронте еще не все реализовано в соответствии с бэком.

## MVP

MVP проекта считается готовым, когда выполнены критичные задачи из обоих направлений:

1. backend поддерживает авторизацию, выдачу заданий, проверку SQL и сохранение прогресса;
2. frontend даёт пользователю полный сценарий: вход, выбор уровня, решение задачи и просмотр результата;
3. карта уровней и прогресс синхронизированы между интерфейсом и сервером.

## Как использовать в трекере

- Если задача относится только к API, данным, SQL-движку или хранению прогресса, её нужно брать из backend roadmap.
- Если задача относится только к экранам, маршрутам, состоянию UI, feedback и аналитическим визуализациям, её нужно брать из frontend roadmap.
- Если фича требует изменений с двух сторон, стоит заводить одну крупную `parent issue` и создавать в ней отдельные backend/frontend `sub-issues`.
